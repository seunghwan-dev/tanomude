# 出張申請 操作マニュアル（汎用サンプル）

本書は架空サンプルである。固有名は一般識別子（自社 / 実験機A・実験機B / Material-X / ABC商事 / 製品X）に抽象化している。

## 1. 概要

出張申請は基幹系（AS-400 グリーンスクリーン）の出張申請画面で行う。
ログイン後、メニューから出張申請画面を開き、必要項目を入力して確定・送信する。
入力値の検証と画面遷移はシステムが行い、確定時に申請レコードが登録される。

## 2. 入力フィールド

出張申請画面の各フィールドの形式と制約を示す。

### 2.1 出張先 (DEST)

半角英大文字の都市コードで入力する。最大12文字。必須項目。
ABC商事との打ち合わせなど、訪問先に応じた都市コードを選ぶ。

### 2.2 出発日・帰着日 (DEPTDATE / RETDATE)

いずれも YYYYMMDD の8桁で入力する。必須項目。
日帰りの場合は帰着日に出発日と同じ値を入力する。

### 2.3 日数 (DAYS)

出発日と帰着日を含む日数を数値で入力する。必須項目。
日数が日付と矛盾する場合は日付から算出した値を正とする。

### 2.4 目的 (PURPOSE)

出張の目的を全角文字列で入力する。最大20文字、超過分は切り捨てる。
製品Xの納入調整、実験機Aの立会検査など、簡潔に記載する。

### 2.5 案件コード (PROJ)

形式は P-### の5文字。直接入力せず F4 プロンプトで一覧から選択する。
不明または形式不正の場合も必ず F4 で正しいコードを選ぶ。

### 2.6 海外区分 (OVRSEA)

海外出張の場合のみ Y を設定する。国内出張では空欄のままとする。

## 3. キー操作

出張申請画面で使用するキーの挙動を示す。

### 3.1 Enter

メニューから申請画面への遷移、入力内容の確定、確定後の申請送信に用いる。

### 3.2 F3

入力を中断し前画面へ戻す。必須項目が空のまま送信できないため、欠落時は F3 で差し戻す。

### 3.3 F4

案件コードのプロンプトを表示し、一覧から選択する。

### 3.4 F9

前回入力した案件コードを再呼び出しする。

### 3.5 FieldExit / Tab

数値フィールドの確定や、海外区分など分岐フィールドへの移動に用いる。

## 4. 標準手順

ログイン後、メニューで Enter を押して出張申請画面を開く。
出張先・出発日・帰着日・日数・目的を順に入力し、F4 で案件コードを選択する。
Enter で入力を確定し、もう一度 Enter で申請を送信する。
必須項目が欠落している場合は値を捏造せず F3 で中断する。
